1. Introduction & Efficiency

An algorithm is a sequence of unambiguous instructions for solving a problem, designed to obtain a required output for any legitimate input in a finite amount of time. It can be viewed as a function f that maps input X to output Y.

Qualities of a Good Algorithm

  • Precision: Input and output should be defined precisely.
  • Unambiguity: Each step should be clear.
  • Effectiveness: It should be the most effective way to solve the problem among alternatives.
  • Language Independence: It should not include specific computer code but be written so it can be implemented in various programming languages.

Complexity Analysis

  • Time Complexity: The amount of computer time required to run an algorithm. It depends on machine speed, input size (n), and the number of operations.
  • Space Complexity: The extra space used by the algorithm beyond the space allocated for the input.

2. Complexity Bounds

Type of Bound Notation Definition & Meaning
Upper Bound Big O (O) T(n) ≤ c · f(n). Describes worst-case behavior.
Lower Bound Omega (Ω) T(n) ≥ c · f(n). Describes best-case behavior.
Tight Bound Theta (Θ) c1f(n) ≤ T(n) ≤ c2f(n). Used for "Exact" algorithms where cost is independent of data values.

Common Growth Rates (Slowest to Fastest)

  • O(1) Constant (e.g., accessing an array element)
  • O(log n) Logarithmic (e.g., Binary Search)
  • O(n) Linear (e.g., Linear Search)
  • O(n log n) Linearithmic (e.g., Merge Sort)
  • O(n2) Quadratic (e.g., Bubble/Selection Sort)
  • O(n3) Cubic (e.g., Matrix Multiplication)
  • O(2n) Exponential (e.g., Recursive Fibonacci)
  • O(n!) Factorial (e.g., TSP Brute Force)

3. Recurrence Relations

Recurrences arise when an algorithm contains recursive calls. The Master Theorem provides a cookbook method for solving them.

Master Theorem for Dividing Functions

For a recurrence: T(n) = aT(n/b) + f(n), where f(n) = Θ(nd)

  • Case 1: If a < bd, then T(n) = Θ(nd)
  • Case 2: If a = bd, then T(n) = Θ(nd logb n)
  • Case 3: If a > bd, then T(n) = Θ(nlogb a)

Master Theorem for Decreasing Functions

For a recurrence: T(n) = aT(n-b) + f(n), where f(n) = Θ(nd)

  • Case 1: If a < 1, then T(n) = Θ(f(n))
  • Case 2: If a = 1, then T(n) = Θ(n · f(n)) or Θ(nd+1)
  • Case 3: If a > 1, then T(n) = Θ(an/b · f(n))

Greatest Common Divisor (Euclid)

// Iterative Implementation
unsigned int GCD_Iterative(unsigned int m, unsigned int n) {
    unsigned int r;
    while (n > 0) {
        r = m % n;
        m = n;
        n = r;
    }
    return m;
}

// Recursive Implementation
unsigned int gcd(unsigned int m, unsigned int n) {
    if (n == 0) return m;
    else return gcd(n, m % n);
}

Matrix Multiplication & Fibonacci

  • Matrix Mult: Efficiency: O(n3)
  • Fibonacci (Iterative): Efficiency: O(n)
  • Fibonacci (Recursive): Efficiency: O(2n/2)
// Basic Matrix Multiplication
void MatrixMult(int n, double A[][Dimen2], double B[][Dimen2], double C[][Dimen2]) {
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++) {
            C[i][j] = 0.0;
            for (int k = 0; k < n; k++)
                C[i][j] = C[i][j] + A[i][k] * B[k][j];
        }
}

// Iterative Fibonacci (Dynamic Programming)
int Fib2(int n) {
    int *f = new int[n + 1];
    f[0] = 0;
    if (n > 0) {
        f[1] = 1;
        for (int i = 2; i <= n; i++)
            f[i] = f[i - 1] + f[i - 2];
    }
    return f[n];
}

// Recursive Fibonacci (Divide and Conquer)
int Fib(int n) {
    if (n <= 1) return n;
    return Fib(n - 1) + Fib(n - 2);
}

Search & Sort Implementations

Searching Algorithms

  • Sequential Search: Worst: O(n) | Best: Ω(1)
  • Binary Search: Efficiency: O(log n)
template <typename DataType>
void seqSearch(int n, DataType *S, DataType x, int& location) {
    location = 1;
    while (location <= n && S[location] != x)
        location++;
    if (location > n)
        location = 0;
}

template <typename DataType>
void binSearch(int n, DataType *S, DataType x, int& location) {
    int low = 0, high = n - 1, mid;
    location = 0;
    while (low <= high && location == 0) {
        mid = (low + high) / 2;
        if (x == S[mid]) location = mid + 1;
        else if (x < S[mid]) high = mid - 1;
        else low = mid + 1;
    }
}

Sorting Algorithms

  • Exchange Sort: Efficiency: O(n2)
  • Merge Sort: Efficiency: Θ(n log n)
// Exchange Sort
template <typename DataType>
void ExchangeSort(int n, DataType *S) {
    DataType tmp;
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++)
            if (S[j] < S[i]) {
                tmp = S[j];
                S[j] = S[i];
                S[i] = tmp;
            }
}

// Merge Sort (Divide and Conquer)
template <typename ElementType>
void mergesort(ElementType *a, ElementType *b, int low, int high) {
    if (low < high) {
        int Mid = (low + high) / 2;
        mergesort(a, b, low, Mid);
        mergesort(a, b, Mid + 1, high);
        merge(a, b, low, Mid, high);
    }
}

template <typename DataType>
void merge(DataType *a, DataType *b, int low, int Mid, int high) {
    int i = low, x = low, y = Mid + 1;
    while ((x <= Mid) && (y <= high)) {
        if (a[x] <= a[y]) { b[i] = a[x]; x++; }
        else { b[i] = a[y]; y++; }
        i++;
    }
    if (x > Mid) for (int k = y; k <= high; k++) { b[i] = a[k]; i++; }
    else for (int k = x; k <= Mid; k++) { b[i] = a[k]; i++; }
    for (int k = low; k <= high; k++) a[k] = b[k];
}

Brute Force & Graphs

Brute Force / Exhaustive Search

  • Naive String Matching: Efficiency: O(nm)
  • Closest Pair (Brute Force): Efficiency: Θ(n2)
// Naive String Matching
int NaiveStringMatch(string T, string P) {
    int i, j;
    for (i = 0; i <= T.length() - P.length(); i++) {
        j = 0;
        while ((j < P.length()) && (P[j] == T[i + j])) {
            j++;
        }
        if (j == P.length()) return i;
    }
    return -1;
}

// Closest Pair
double ClosestPairMod(MyPT *Ar, int size, int& Pair1, int& Pair2) {
    double d = 2.0e10, Dx, Dy;
    for (int i = 0; i < size; i++) {
        for (int j = i + 1; j < size; j++) {
            Dx = (double)Ar[i].x - Ar[j].x;
            Dy = (double)Ar[i].y - Ar[j].y;
            double dist = sqrt((Dx * Dx) + (Dy * Dy));
            if (d > dist) {
                d = dist;
                Pair1 = i;
                Pair2 = j;
            }
        }
    }
    return d;
}

Graph Traversal

  • DFS & BFS: Θ(V2) for matrix, Θ(|V| + |E|) for adjacency list.
// Depth First Search (DFS)
void DirectedUnweightedGraph::DFSRec(int vertex, vector<bool>& visited) {
    if (visited[vertex]) return;
    cout << vertex << " ";
    visited[vertex] = true;
    for (int i = 0; i < adjacencyList[vertex].size(); i++) {
        int dest = adjacencyList[vertex][i];
        if (!visited[dest]) DFSRec(dest, visited);
    }
}

// Breadth First Search (BFS)
void DirectedUnweightedGraph::BFS() {
    vector<bool> visited(numberOfVertices, false);
    queue<int> q;
    q.push(0); // Start at vertex 0
    visited[0] = true;
    while (!q.empty()) {
        int vertex = q.front();
        q.pop();
        cout << vertex << " ";
        for (int i = 0; i < adjacencyList[vertex].size(); i++) {
            int dest = adjacencyList[vertex][i];
            if (!visited[dest]) {
                q.push(dest);
                visited[dest] = true;
            }
        }
    }
}

Glossary of Key Terms

  • Adjacency Matrix: A 2D array used to represent a graph where A[i][j] = 1 if there is an edge between i and j.
  • Backtracking: A method of searching for a solution by exploring paths and returning when a dead end is reached (used in DFS).
  • Basic Operation: The most frequent operation in an algorithm (e.g., comparison in sorting) used for asymptotic analysis.
  • Brute Force: A straightforward approach based directly on the problem statement.
  • Divide and Conquer: A strategy that breaks a problem into sub-problems, solves them recursively, and combines the results.
  • Exhaustive Search: A brute force solution for combinatorial problems (permutations, subsets).
  • Hamiltonian Circuit: A path that visits every vertex in a graph exactly once and returns to the start.
  • Knapsack Problem: Given weights and values of items, find the most valuable subset that fits in a weight limit W.
  • Traveling Salesman Problem (TSP): Finding the shortest tour through n cities.

Practice Quiz

Questions:

  1. What is the worst-case complexity of Binary Search?
  2. True or False: Big-O notation is used to describe the lower bound of an algorithm.
  3. Which data structure is used for Breadth-First Search (BFS)?
  4. What is the recurrence relation for Merge Sort?
  5. In the Master Theorem for dividing functions, what is the complexity if a > bd?
  6. How many permutations are there in an exhaustive search for a TSP with n cities?
  7. What is the basic operation in the Sieve of Eratosthenes?
  8. What is the difference between O(n) and Θ(n)?

Answers:

  • A1: O(log2 n)
  • A2: False. Big-O is for the upper bound (worst case); Omega (Ω) is for the lower bound.
  • A3: A Queue.
  • A4: W(n) = 2W(n/2) + n - 1
  • A5: T(n) = Θ(nlogb a)
  • A6: (n-1)! or (n-1)! / 2
  • A7: Marking composite numbers (setting an array index to 0).
  • A8: O(n) is an upper bound (the algorithm takes at most n time); Θ(n) is a tight bound (the algorithm takes exactly proportional to n time regardless of data).
