We live in a world of finite resources. Speed isn't just a "nice to have," it's critical.

Time (CPU)

How much memory are we using?

Recursive vs Iterative

Case Study: Calculating the n-th Fibonacci number.

// Recursive (Slow) - O(2^n) int fib(int n) { if (n <= 1) return n; return fib(n-1) + fib(n-2); }

The Reality Check:

For $n=100$, the iterative version takes 100 ns. The recursive version would take 13 days!

Chapter 2: Complexity Bounds

We use Asymptotic Analysis to focus on growth, ignoring machine-specific constants.

NotationMeaningAnalogy
$O(f(n))$Upper Bound (Worst Case)"It won't take longer than..."
$\Omega(f(n))$Lower Bound (Best Case)"It will take at least..."
$\Theta(f(n))$Tight Bound (Exact)"It grows exactly like..."

Growth Rate Rankings

From "Lightning Fast" to "Practically Impossible":

$O(1) < O(\log n) < O(n) < O(n \log n) < O(n^2) < O(2^n) < O(n!)$

The Laws of Big-O

1. Drop Constants

$O(3n^2 + 5) \rightarrow O(n^2)$

2. Sum Rule

$O(n + n^2) \rightarrow O(n^2)$

3. Product Rule

$O(n \cdot \log n) \rightarrow O(n \log n)$

(Usually found in nested loops)

Chapter 3: Brute Force

The "just do it" approach. Based directly on the problem statement.

Naive String Search

Check every index $i$ in Text $T$ for Pattern $P$.

Complexity: $O(n \cdot m)$

for(i=0; i <= n-m; i++) { j=0; while(j < m && P[j] == T[i+j]) j++; if(j == m) return i; }

3.3 Exhaustive Search

Checking every possible combination. Great for small $n$, lethal for large $n$.

TSP Problem

Generate $(n-1)!$ permutations.

$O(n!)$

Knapsack

Check all $2^n$ subsets.

$O(2^n)$

💡 Note: For $n=40$, Exhaustive Search might take longer than the age of the universe.

Chapter 4: The Master Theorem

The shortcut for solving recurrences of form $T(n) = aT(n/b) + \Theta(n^d)$.

Comparing $a$ and $b^d$:

  • Case 1 ($a < b^d$): $T(n) = \Theta(n^d)$
  • Case 2 ($a = b^d$): $T(n) = \Theta(n^d \log n)$
  • Case 3 ($a > b^d$): $T(n) = \Theta(n^{\log_b a})$

Example: Merge Sort $\rightarrow T(n) = 2T(n/2) + n$

$a=2, b=2, d=1$. Since $2 = 2^1$, Result is $\Theta(n \log n)$.

Chapter 5: Graph Traversal (DFS)

Depth First Search

  • Motto: "Go deep as fast as possible."
  • Data Structure: Stack (Recursion)
  • Edges: Tree + Back edges

Graph Traversal (BFS)

Breadth First Search

  • Motto: "Look at all neighbors first."
  • Data Structure: Queue
  • Key Use: Finding the Shortest Path.
Efficiency for both: $\Theta(V + E)$ using Adjacency Lists.
